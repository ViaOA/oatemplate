JSON servlet Help

The OA JSON Servlet is used to access the model's data objects, and metadata about the classes.

This will allow querying of the data that is maintained within an application and have it returned in a json format. All queries specify the name of the data class, along with the object Id to return, or a query. A query is similar to a database query, and is based on the object model. Property paths are used to build a reference from one object to another.

Usage

http://hostname:port/servlet/json
to display this help information

Example:

http://hostname:port/servlet/json?class=Application&query="ApplicationStatus.type!=5"
in this example, the list is all of the applications that are not running.

HTTP Query parameters

/servlet/json root context
? begin of http query
class|c name of object class that will be returned (case sensitive)

next, use an id or query for the object(s) that are requested.
id= the id of the class instance to return
query= the query to use (see below)
all if the return value is a collection, then show all of the data, otherwise (default) only list the Id values. describe used to describe information (metadata) about the class.


Query

Use the application object model to get the name of the data objects (class).


Examples



servlet/json?c=Site&query="environments.silos.servers.applications.applicationType.code like 'fix*'"

servlet/json?c=Application&query="applicationType.code like 'fix*'"

servlet/json?c=Application&query="applicationStatus.type=5"

servlet/json?c=Application&query="(id<398 OR id > 400) AND applicationType.code like 'fix*'"
servlet/json?c=Application&query="id=398"

/json?c=Application&describe
